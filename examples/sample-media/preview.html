日本語UIの呼吸を壊さないための
DESIGN.md と mixed-script 設計

この preview は、記事メディア向けの contract を置いたときに、本文、見出し、URL、 日本語 + English product name の混在がどう見えるかを確認するための最小サンプルです。

日本語UIの品質は、色や角丸だけでは決まりません。特に長文画面では、 Design Contract の有無が、本文の呼吸、見出しの折り返し、 URL のはみ出し、和欧混植の違和感にそのまま出ます。

たとえば「AI builder 向けの design system」という見出しをそのまま置くと、英語部分だけが 浮いたり、モバイルで苦しい位置で折れたりします。だからこそ、日本語本文の line-height、 mixed-script 見出し、URL overflow の扱いを別々に contract する必要があります。

キャプション例: 画像は文章の流れを切るためではなく、読書のリズムを切り替えるために置く。

まず contract に戻すべきもの

最初から完璧な design system を書く必要はありません。まずは「壊れやすい箇所」を contract に戻します。 典型例は、長文本文、mixed-script 見出し、長い URL、引用、キャプション、そして narrow width です。

きれいなスクリーンショットより先に、長文を実際に読めることを確認する。 これが media profile の最優先です。

URL と長い語の挙動

たとえば次のURLが記事中に入っても、段落全体の可読性を壊さないことが重要です。 https://example.com/design-contracts/japanese-ui/mixed-script-heading-and-long-url-overflow-check

本文の字間は盛らない

日本語の本文では、見出しやラベルと違って、字間を強く開くほど読みやすくなるとは限りません。 まずは本文の行間で呼吸を作り、字間は保守的に保つほうが安定します。
